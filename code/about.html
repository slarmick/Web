🚀 Функциональность ЛР-7 (Асинхронные очереди)

Основные возможности системы очередей:

  • Двойная интеграция - работа с RabbitMQ и Kafka в одном проекте
  • Двухуровневая архитектура - основные очереди и очереди ошибок
  • Асинхронная обработка - отправка данных без блокировки пользователя
  • Отказоустойчивость - продолжение работы при недоступности очередей
  • Мониторинг в реальном времени - Queue Dashboard со статистикой
  • Логирование операций - детальное отслеживание всех сообщений
  • Автоматическое восстановление - обработка ошибок с перемещением в очередь ошибок

Системы очередей:

🐇 RabbitMQ (порт 5672)

Тип: Message broker (AMQP protocol)

Применение: Основная очередь для надежной доставки сообщений

Особенности: Гарантированная доставка, подтверждения, persistence

Панель управления: http://localhost:15672

🦊 Apache Kafka (порт 9092)

Тип: Distributed streaming platform

Применение: Обработка потоков данных, логгирование

Особенности: Высокая пропускная способность, распределенность

Структура: Топики (lab7_main_topic, lab7_error_topic)

📊 Queue Dashboard

Назначение: Мониторинг и управление очередями

Функции: Статистика подключений, просмотр сообщений, управление

Доступ: http://localhost:8080/queue_dashboard.php

Технологии: PHP, HTML5, CSS3, JavaScript

Архитектура обработки данных:

1. Пользователь заполняет форму → данные валидируются

2. Сохранение в MySQL → основное реляционное хранилище

3. Кэширование в Redis → быстрый доступ, сессии, статистика

4. Отправка в RabbitMQ → гарантированная асинхронная обработка

5. Отправка в Kafka → потоковая обработка и аналитика

6. Запись в файл → резервное хранение для совместимости

7. Ответ пользователю → перенаправление на страницу результатов

Как использовать:

  1. Запустите проект: docker-compose up -d --build
  2. Откройте главную страницу: http://localhost:8080
  3. Перейдите к форме регистрации и заполните её
  4. Просмотрите Queue Dashboard: http://localhost:8080/queue_dashboard.php
  5. Мониторьте RabbitMQ: http://localhost:15672 (guest/guest)
  6. Проверьте логи Kafka: docker logs lab7_kafka --tail 20

Команды для работы:

# Запуск проекта
docker-compose up -d --build

# Просмотр логов
docker logs lab7_php --tail 20
docker logs lab7_kafka --tail 10

# Проверка очередей
curl http://localhost:15672/api/queues

# Тестовый worker (демо)
docker exec -it lab7_php php /var/www/html/queue_worker.php
