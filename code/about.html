🚀 Функциональность ЛР-6

Основные возможности:

  • Redis Integration - работа с ключ-значение хранилищем через REST API
  • Elasticsearch Integration - полнотекстовый поиск и индексация документов
  • ClickHouse Integration - аналитические запросы и обработка больших данных
  • Multi-Container Architecture - Redis, Elasticsearch, ClickHouse в отдельных контейнерах
  • HTTP API Communication - взаимодействие с БД через Guzzle HTTP клиент
  • Client Factory Pattern - унифицированный подход к работе с различными API
  • Redis Dashboard - веб-интерфейс для мониторинга и управления Redis
  • 30 Use Cases - реализация различных сценариев использования NoSQL БД

Нереляционные базы данных:

🔴 Redis (порт 6379)

Тип: In-memory key-value store

Применение: Кэширование, сессии, очереди, реальное время

Примеры использования: Пользователи, чаты, сессии, уведомления

🔍 Elasticsearch (порт 9200)

Тип: Поисковый движок и аналитическая платформа

Применение: Полнотекстовый поиск, логи, аналитика

Примеры использования: Товары, книги, фильмы, новости, отзывы

⚡ ClickHouse (порт 8123)

Тип: Колоночная аналитическая СУБД

Применение: Аналитика больших данных, OLAP-запросы

Примеры использования: Погода, транспорт, сенсоры, транзакции, логи

Примеры тематических заданий:

Пользователи и сессии - Redis для хранения пользовательских данных и сессий

Поиск товаров и книг - Elasticsearch для полнотекстового поиска

Аналитика транзакций - ClickHouse для аналитических отчетов

Мониторинг в реальном времени - Redis для дашбордов и уведомлений

Обработка логов - Elasticsearch для анализа логов приложения

Как использовать:

  1. Запустите проект: docker-compose up -d --build
  2. Откройте в браузере: http://localhost:8080
  3. Перейдите на Redis Dashboard: http://localhost:8080/redis-dashboard.php
  4. Используйте API endpoints для работы с базами данных:
    • Redis: REST API через порт 7379
    • Elasticsearch: REST API через порт 9200
    • ClickHouse: HTTP интерфейс через порт 8123
